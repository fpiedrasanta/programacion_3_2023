Programación 3

¿Que es la programación?

La programación es el proceso de diseñar, escribir, probar y mantener el código fuente de un software. Consiste en la creación de instrucciones que una computadora puede seguir para llevar a cabo una tarea específica. Estas instrucciones, escritas en lenguajes de programación, permiten a los programadores controlar el comportamiento de una computadora y desarrollar aplicaciones, programas, sitios web y otros sistemas informáticos.

Programa de la materia

Unidad 1

HTML/CSS

Tabla de contenidos:

  • Prototipado y conceptos de HTML
  • Conceptos de CSS
  • Box modeling
  • Flexbox y Grids
  • Animaciones
  • Frameworks para estilos: Bootstrap

Fundamentos:

  1. Creación y edición de contenido web: HTML es el lenguaje fundamental para crear y editar contenido en la web. Aunque existen plataformas y sistemas de gestión de contenido (CMS) que ofrecen interfaces visuales para construir sitios web, tener conocimientos básicos de HTML te permite tener un mayor control y personalización sobre tu contenido.
  2. Compatibilidad y accesibilidad: HTML es un lenguaje universalmente compatible. Todos los navegadores web son capaces de interpretar y mostrar correctamente el código HTML, lo que garantiza que tu contenido sea accesible para una amplia audiencia en diferentes dispositivos y plataformas.
  3. Comunicación con desarrolladores y diseñadores: Comprender el código HTML les permite colaborar de manera más estrecha, transmitir tus ideas y requerimientos de manera más precisa, y comprender mejor las limitaciones y posibilidades técnicas de un proyecto.
  4. Solución de problemas y personalización: A veces, es posible que encuentres problemas en el contenido de tu sitio web o desees personalizar ciertos aspectos más allá de las opciones proporcionadas por las herramientas visuales. Con conocimientos de HTML, puedes inspeccionar y comprender el código existente, identificar y corregir problemas, así como personalizar elementos específicos según tus necesidades.

Unidad 2

JavaScript

Tabla de contenidos:

  • Introducción a Javascript
  • Sintaxis y variables
  • Funciones y objetos
  • Arrays y funciones de orden superior
  • DOM y Eventos
  • Local Storage
  • JSON
  • Async/Await, fetch y Ajax

Fundamentos:

  1. Interactividad en el lado del cliente: JavaScript permite agregar interactividad y dinamismo a las páginas web. Puedes crear funciones y eventos que respondan a las acciones del usuario.
  2. Manipulación del DOM: El Document Object Model (DOM) es una representación en memoria de la estructura de una página web. JavaScript permite acceder y manipular el DOM para realizar cambios en tiempo real en los elementos de una página.
  3. Validación de formularios: JavaScript es útil para realizar validaciones en formularios web antes de enviar los datos al servidor.
  4. Desarrollo de aplicaciones web completas: Con el uso de JavaScript y frameworks populares como React, Angular o Vue.js, es posible desarrollar aplicaciones web completas con una interfaz de usuario sofisticada y una experiencia de usuario fluida.
  5. Comunicación con el servidor: JavaScript se utiliza para realizar solicitudes asíncronas al servidor a través de AJAX (Asynchronous JavaScript and XML). Esto permite cargar y enviar datos en segundo plano sin tener que recargar toda la página. También se utiliza para consumir y manipular APIs (Application Programming Interfaces) de terceros, lo que brinda la capacidad de integrar servicios externos en tus aplicaciones web.
  6. Popularidad y demanda laboral: JavaScript es uno de los lenguajes de programación más utilizados en la actualidad. Tener habilidades en JavaScript aumenta tus oportunidades laborales, ya que hay una alta demanda de desarrolladores web que conocen este lenguaje.

Unidad 3

API REST

Tabla de contenidos:

  • Introducción a la arquitectura cliente – servidor
  • Sintaxis y variables de un lenguaje de backend
  • Control de flujos
  • Ciclos e iteraciones
  • Funciones
  • Clases y objetos

Fundamentos:

  1. Comunicación entre aplicaciones: Las API REST permiten que diferentes aplicaciones se comuniquen y compartan datos de manera eficiente.
  2. Arquitectura escalable y flexible: REST es un estilo arquitectónico que fomenta la escalabilidad y la modularidad.
  3. Acceso a datos y funcionalidades: Al exponer una API REST, puedes permitir que otros desarrolladores o servicios accedan y utilicen los datos y funcionalidades de tu aplicación de manera controlada.
  4. Interoperabilidad y estándares: REST se basa en protocolos y estándares web ampliamente adoptados, como HTTP, URIs (Uniform Resource Identifiers) y formatos de datos como JSON o XML.
  5. Seguridad y control de acceso: Al diseñar una API REST, puedes implementar mecanismos de autenticación y autorización para controlar el acceso a los recursos y las operaciones de tu aplicación.

Unidad 4

Seguridad

Tabla de contenidos:

  • Concepto de ORM
  • Entity Framework (code first y database first)
  • Concepto de Entity y Model
  • JWT (Json Web Token)

Fundamentos:

  1. Persistencia de datos: Las aplicaciones web suelen requerir almacenamiento y gestión de datos.
  2. Operaciones de lectura y escritura: Las aplicaciones necesitan acceder y manipular datos, realizar operaciones de lectura (consultas) y escritura (inserción, actualización y eliminación).
  3. Eficiencia y optimización: Un acceso eficiente a los datos es esencial para el rendimiento y la escalabilidad de una aplicación web.
  4. Protección de datos sensibles: Las aplicaciones web manejan datos sensibles, como información personal de los usuarios, contraseñas, datos financieros, etc. Es fundamental implementar medidas de seguridad para proteger estos datos.
  5. Autenticación y autorización: Es importante asegurarse de que solo los usuarios autorizados puedan acceder a ciertos datos y realizar ciertas acciones en la aplicación.

Método de evaluación

Se realizarán 2 instancias de exámenes parciales orales abordando los contenidos desarrollados en clase y viendo el código desarrollado (code-review).

  • Una nota práctica grupal, por cada instancia de examen, que surge del código.
  • Una nota teórica individual, por cada instancia de examen, que surge de la defensa y explicación del código.

En total se tendrá 4 notas (2 grupales y 2 individuales) al finalizar el año.

Además, se prevé una nota final integradora, donde se evaluará el proyecto final funcionando.

CONDICIÓN DE REGULARIDAD

  • Asistir al 80% (ochenta porciento) de las clases
  • Aprobar todas las evaluaciones parciales o sus respectivos recuperatorios con un mínimo de 4 (cuatro) puntos equivalente al (60%).

CONDICIÓN DE PROMOCIÓN

  • Asistir al 80% (ochenta porciento) de las clases
  • Aprobar todas las evaluaciones parciales con un mínimo de 7 (siete) puntos y promedio de 8 (ocho) puntos.
  • Aprobar el examen integrador final con un mínimo de 7 (siete) puntos.
