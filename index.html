Programación 3

¿Que es la programación?

La programación es el proceso de diseñar, escribir, probar y mantener el código fuente de un software. Consiste en la creación de instrucciones que una computadora puede seguir para llevar a cabo una tarea específica. Estas instrucciones, escritas en lenguajes de programación, permiten a los programadores controlar el comportamiento de una computadora y desarrollar aplicaciones, programas, sitios web y otros sistemas informáticos.

La programación implica la utilización de algoritmos, que son secuencias de pasos lógicos que resuelven un problema en particular. Estos algoritmos se traducen al lenguaje de programación elegido por el programador, que puede ser de alto nivel como Python, Java o C++, o de bajo nivel como el lenguaje ensamblador.

Los programadores utilizan diferentes herramientas de desarrollo, como editores de texto y entornos de desarrollo integrados (IDE), para escribir y organizar el código fuente. Una vez que el código ha sido escrito, se debe compilar o interpretar para convertirlo en un formato ejecutable que la computadora pueda entender y ejecutar.

La programación requiere habilidades lógicas y analíticas, así como la capacidad de resolver problemas de manera estructurada. Los programadores también deben tener un buen entendimiento de las estructuras de datos, los algoritmos y las mejores prácticas de programación para escribir un código eficiente y mantenible.

La programación es una habilidad fundamental en la era digital, ya que está presente en casi todos los aspectos de nuestra vida cotidiana. Desde los sistemas operativos en nuestros dispositivos móviles, hasta las aplicaciones web y de escritorio que utilizamos, todo se basa en el código escrito por programadores. A medida que la tecnología avanza, la demanda de programadores cualificados continúa creciendo.

Unidad 1

HTML/CSS

Tabla de contenidos:

  • Prototipado y conceptos de HTML
  • Conceptos de CSS
  • Box modeling
  • Flexbox y Grids
  • Animaciones
  • Frameworks para estilos: Bootstrap

Fundamentos:

  1. Creación y edición de contenido web: HTML es el lenguaje fundamental para crear y editar contenido en la web. Aunque existen plataformas y sistemas de gestión de contenido (CMS) que ofrecen interfaces visuales para construir sitios web, tener conocimientos básicos de HTML te permite tener un mayor control y personalización sobre tu contenido. Puedes ajustar la estructura de tus páginas, agregar enlaces, imágenes, videos y otros elementos multimedia, dar formato al texto, entre otros aspectos.
  2. Compatibilidad y accesibilidad: HTML es un lenguaje universalmente compatible. Todos los navegadores web son capaces de interpretar y mostrar correctamente el código HTML, lo que garantiza que tu contenido sea accesible para una amplia audiencia en diferentes dispositivos y plataformas. Además, el uso adecuado de las etiquetas semánticas en HTML ayuda a mejorar la accesibilidad web, permitiendo a las personas con discapacidades utilizar tecnologías de asistencia para navegar y comprender el contenido.
  3. Comunicación con desarrolladores y diseñadores: Si trabajas en el ámbito digital, ya sea como diseñador, desarrollador, gestor de contenido o en marketing, tener conocimientos básicos de HTML te permite comunicarte de manera más efectiva con profesionales técnicos. Comprender el código HTML les permite colaborar de manera más estrecha, transmitir tus ideas y requerimientos de manera más precisa, y comprender mejor las limitaciones y posibilidades técnicas de un proyecto.
  4. Solución de problemas y personalización: A veces, es posible que encuentres problemas en el contenido de tu sitio web o desees personalizar ciertos aspectos más allá de las opciones proporcionadas por las herramientas visuales. Con conocimientos de HTML, puedes inspeccionar y comprender el código existente, identificar y corregir problemas, así como personalizar elementos específicos según tus necesidades.

En resumen, saber HTML te proporciona habilidades fundamentales para crear, editar y personalizar contenido web, asegurando la compatibilidad y accesibilidad de tus páginas. Además, facilita la colaboración con profesionales técnicos y te permite solucionar problemas y personalizar elementos en tu sitio web.

HTML es el lenguaje de marcado estándar utilizado para crear páginas web. Es la base fundamental de cualquier sitio web, ya que define la estructura y el contenido de una página. Con conocimientos en HTML, puedes diseñar y desarrollar tus propias páginas web desde cero, controlando la apariencia y la organización de los elementos en la página.

Unidad 2

JavaScript

Tabla de contenidos:

  • Introducción a Javascript
  • Sintaxis y variables
  • Funciones y objetos
  • Arrays y funciones de orden superior
  • DOM y Eventos
  • Local Storage
  • JSON
  • Async/Await, fetch y Ajax

Fundamentos:

  1. Interactividad en el lado del cliente: JavaScript permite agregar interactividad y dinamismo a las páginas web. Puedes crear funciones y eventos que respondan a las acciones del usuario, como hacer clic en un botón, desplazarse por una página o enviar un formulario. Esto te permite crear experiencias interactivas y en tiempo real para tus usuarios.
  2. Manipulación del DOM: El Document Object Model (DOM) es una representación en memoria de la estructura de una página web. JavaScript permite acceder y manipular el DOM para realizar cambios en tiempo real en los elementos de una página. Puedes agregar, modificar o eliminar elementos, cambiar estilos, contenido y atributos, lo que brinda un mayor control y dinamismo en la presentación de tus páginas.
  3. Validación de formularios: JavaScript es útil para realizar validaciones en formularios web antes de enviar los datos al servidor. Puedes verificar que se completen los campos obligatorios, validar el formato de datos como direcciones de correo electrónico o números de teléfono, y proporcionar retroalimentación instantánea al usuario si se detecta algún error.
  4. Desarrollo de aplicaciones web completas: Con el uso de JavaScript y frameworks populares como React, Angular o Vue.js, es posible desarrollar aplicaciones web completas con una interfaz de usuario sofisticada y una experiencia de usuario fluida. Estas herramientas facilitan la creación de componentes reutilizables, gestión de estado y enrutamiento, lo que permite construir aplicaciones web modernas y escalables.
  5. Comunicación con el servidor: JavaScript se utiliza para realizar solicitudes asíncronas al servidor a través de AJAX (Asynchronous JavaScript and XML). Esto permite cargar y enviar datos en segundo plano sin tener que recargar toda la página. También se utiliza para consumir y manipular APIs (Application Programming Interfaces) de terceros, lo que brinda la capacidad de integrar servicios externos en tus aplicaciones web.
  6. Popularidad y demanda laboral: JavaScript es uno de los lenguajes de programación más utilizados en la actualidad. Tener habilidades en JavaScript aumenta tus oportunidades laborales, ya que hay una alta demanda de desarrolladores web que conocen este lenguaje. Además, la comunidad de JavaScript es activa y ofrece una amplia variedad de recursos, bibliotecas y frameworks que facilitan el desarrollo web.

En resumen, JavaScript es esencial para agregar interactividad, manipular el contenido de las páginas web, validar formularios y desarrollar aplicaciones web completas. Su popularidad y demanda laboral hacen que sea un lenguaje importante para los desarrolladores web.

Unidad 3

API REST

Tabla de contenidos:

  • Introducción a la arquitectura cliente – servidor
  • Sintaxis y variables de un lenguaje de backend
  • Control de flujos
  • Ciclos e iteraciones
  • Funciones
  • Clases y objetos

Fundamentos:

  1. Comunicación entre aplicaciones: Las API REST permiten que diferentes aplicaciones se comuniquen y compartan datos de manera eficiente. Al construir una API REST, puedes definir puntos finales (endpoints) que representan diferentes recursos y operaciones que se pueden realizar en esos recursos. Esto facilita la integración de tu aplicación con otras aplicaciones o servicios, ya sea en la misma organización o de terceros.
  2. Arquitectura escalable y flexible: REST es un estilo arquitectónico que fomenta la escalabilidad y la modularidad. Al seguir las mejores prácticas de diseño REST, puedes crear APIs que sean fáciles de mantener, extender y escalar a medida que crecen las necesidades de tu aplicación. La separación clara entre el cliente y el servidor, la utilización de verbos HTTP para definir acciones y el uso de formatos de datos comunes como JSON hacen que las APIs REST sean flexibles y compatibles con diferentes tecnologías y plataformas.
  3. Acceso a datos y funcionalidades: Al exponer una API REST, puedes permitir que otros desarrolladores o servicios accedan y utilicen los datos y funcionalidades de tu aplicación de manera controlada. Esto puede abrir nuevas oportunidades de negocio, como la creación de aplicaciones móviles, integración con servicios externos o la posibilidad de desarrollar un ecosistema de complementos y extensiones para tu aplicación.
  4. Interoperabilidad y estándares: REST se basa en protocolos y estándares web ampliamente adoptados, como HTTP, URIs (Uniform Resource Identifiers) y formatos de datos como JSON o XML. Esto facilita la interoperabilidad entre diferentes sistemas y lenguajes de programación. Al seguir las convenciones y estándares REST, puedes asegurarte de que tu API sea compatible con una amplia gama de clientes y servicios.
  5. Seguridad y control de acceso: Al diseñar una API REST, puedes implementar mecanismos de autenticación y autorización para controlar el acceso a los recursos y las operaciones de tu aplicación. Esto te permite proteger tus datos y funcionalidades, garantizando que solo los usuarios autorizados puedan acceder a ellos.

En resumen, la programación de API REST es importante en el desarrollo web porque permite la comunicación entre aplicaciones, promueve una arquitectura escalable y flexible, facilita el acceso a datos y funcionalidades, fomenta la interoperabilidad y proporciona opciones de seguridad y control de acceso. Aprender a diseñar y desarrollar API REST te permite construir sistemas más robustos, interoperables y conectados en el ecosistema web.

Unidad 4

Seguridad

Tabla de contenidos:

  • Concepto de ORM
  • Entity Framework (code first y database first)
  • Concepto de Entity y Model
  • JWT (Json Web Token)

Fundamentos:

  1. Persistencia de datos: Las aplicaciones web suelen requerir almacenamiento y gestión de datos. Pueden utilizar bases de datos relacionales, bases de datos NoSQL, sistemas de archivos u otros medios de almacenamiento. El acceso adecuado a estos datos es esencial para el correcto funcionamiento de la aplicación.
  2. Operaciones de lectura y escritura: Las aplicaciones necesitan acceder y manipular datos, realizar operaciones de lectura (consultas) y escritura (inserción, actualización y eliminación). El acceso a datos se realiza a través de consultas o comandos específicos, utilizando lenguajes de consulta como SQL (Structured Query Language) para bases de datos relacionales, o métodos y funciones proporcionados por bibliotecas y frameworks para bases de datos NoSQL.
  3. Eficiencia y optimización: Un acceso eficiente a los datos es esencial para el rendimiento y la escalabilidad de una aplicación web. Esto implica diseñar una estructura de datos adecuada, indexación eficiente, optimización de consultas y uso de técnicas de caché para reducir la carga en la base de datos.
  4. Protección de datos sensibles: Las aplicaciones web manejan datos sensibles, como información personal de los usuarios, contraseñas, datos financieros, etc. Es fundamental implementar medidas de seguridad para proteger estos datos, como cifrado de datos en reposo y en tránsito, almacenamiento seguro de contraseñas y protección contra ataques de inyección SQL.
  5. Autenticación y autorización: Es importante asegurarse de que solo los usuarios autorizados puedan acceder a ciertos datos y realizar ciertas acciones en la aplicación. La autenticación verifica la identidad del usuario, mientras que la autorización controla los permisos y privilegios de cada usuario en función de su rol o nivel de acceso.

El acceso a datos y la seguridad están estrechamente relacionados en el desarrollo de aplicaciones web. Un acceso seguro a los datos garantiza la integridad, confidencialidad y disponibilidad de la información, protegiendo tanto a los usuarios como a la aplicación en sí. El diseño adecuado de la arquitectura de datos y la implementación de medidas de seguridad robustas son fundamentales para construir aplicaciones web confiables y protegidas.

Método de evaluación

Se realizarán 2 instancias de exámenes parciales orales abordando los contenidos desarrollados en clase, viendo el código desarrollado (code-review).

  • Una nota práctica grupal, por cada instancia de examen, que surge del código.
  • Una nota teórica individual, por cada instancia de examen, que surge de la defensa y explicación del código.

En total se tendrá 4 notas al finalizar el año.

Además, se prevé una nota final integradora, donde se evaluará el proyecto final funcionando.

CONDICIÓN DE REGULARIDAD

  • Asistir al 80% de las clases
  • Aprobar todas las evaluaciones parciales o sus respectivos recuperatorios con un mínimo de 4 puntos equivalente al (60%).

CONDICIÓN DE PROMOCIÓN

  • Asistir al 80% de las clases
  • Aprobar todas las evaluaciones parciales con un mínimo de 7 puntos y promedio de 8 puntos.
  • Aprobar el examen integrador final con un mínimo de 7 puntos.
